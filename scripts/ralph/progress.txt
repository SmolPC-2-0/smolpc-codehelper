## Codebase Patterns
- spawn.rs composable primitives: spawn_engine, wait_for_healthy, check_running_policy, shutdown_and_wait, kill_stale_processes, with_spawn_lock
- RunningHostPolicyDecision has 4 variants: Reuse, Restart, SpawnNeeded, Reject(String) — SpawnNeeded means no engine is running
- connect_or_spawn is now a convenience wrapper calling the primitives — supervisor will call them directly
- resolve_host_binary is pub(crate) — accessible within the crate for the supervisor's spawn logic
- EngineClient::new is pub(crate), token management via load_or_create_token in crate::token
- EngineConnectOptions is the spawn config struct — supervisor will construct this from its own StartupConfig
- Spawn lock (acquire_spawn_lock) is private; use with_spawn_lock for the public API
- smolpc-engine-host has a pre-existing test failure (dev_workspace_lib_roots path mismatch) — not a regression gate
- EngineLifecycleState lives in apps/codehelper/src-tauri/src/engine/mod.rs — tagged enum with serde(tag="state", rename_all="snake_case")
- EngineCommand uses oneshot::Sender<Result<(), String>> for request-response pattern
- StartupConfig wraps RuntimeModePreference, StartupMode from smolpc_engine_client
- smolpc-code-helper lib tests have pre-existing compilation errors (missing tower crate) — not a regression gate
- EngineSupervisorHandle uses mpsc::Sender + watch::Receiver + Arc<Mutex<Option<EngineClient>>> for cached client
- Handle methods use oneshot channels for request-response (ensure_started, set_runtime_mode, shutdown)
- refresh_status() is fire-and-forget (no oneshot response)
- spawn_client_cache_watcher() must be called after handle creation to keep cached_client in sync
- EngineSupervisor is generic over R: Runtime — needs AppHandle<R> for Tauri event emission
- Supervisor broadcasts state via watch::Sender AND emits Tauri events via app_handle.emit("engine-state-changed", &state)
- Supervisor uses a separate watch::Sender<Option<EngineClient>> (client_tx) to feed handle's cached_client watcher
- PID check uses Windows API (OpenProcess/GetExitCodeProcess) — no external crate needed
- Restart policy: 3 restarts within 5 minutes with backoff [1s, 2s, 4s] → Failed after limit
- EngineClient::new is now pub (was pub(crate)) — supervisor constructs clients directly
- load_or_create_token is now pub and re-exported from smolpc_engine_client — supervisor uses it for token regeneration
- do_spawn_sequence resolves paths via app_handle.path() + env vars, same fallback chain as resolve_client in inference.rs
- Supervisor path resolution uses crate::app_paths for bundled resource dir normalization
- resolve_host_binary_path and resolve_models_dir are duplicated in supervisor.rs (from inference.rs) — future cleanup candidate
- Supervisor is wired in Builder::setup() with tokio::spawn(supervisor.run()) — app.manage(handle) registers it as Tauri state
- Exit handler: supervisor.shutdown() is primary, InferenceState.shutdown_engine() is fallback during migration
- Both InferenceState and EngineSupervisorHandle coexist as managed state until US-009 deletes InferenceState
- When calling a Tauri command function directly (not via invoke), you can't pass tauri::State — use &EngineSupervisorHandle directly instead
- For non-blocking engine access (cancel, memory pressure), use supervisor.get_client_if_ready() — never wait 60s for a cancel operation
- After US-009: InferenceState is deleted — all engine access goes through EngineSupervisorHandle exclusively
- Exit shutdown path: supervisor.shutdown() → force_kill_engine() (no more InferenceState fallback)
- EngineLifecycleState TS type is a discriminated union on `state` field — use `state.state` switch for exhaustive handling
- updateLifecycleState() maps supervisor events to existing readiness/engineHealthy/error fields — new consumers can use lifecycleState directly
- Use `eslint-disable-next-line svelte/prefer-svelte-reactivity` for non-reactive Date usage in .svelte.ts files
- Frontend receives supervisor state via `listen('engine-state-changed')` in App.svelte onMount — no health polling
- Banner display uses `inferenceStore.lifecycleState.state` discriminated union (crashed/failed), not engineHealthy boolean
